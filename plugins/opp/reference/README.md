# reference — методический канон

Плоские справочные знания продукта: каркас из 11 уровней (общая «линза»), метод ОБЭ (обоснование
бизнес-эффекта), стандарты формулировок и письма, метод требований и триада решений.

Эти знания всегда в контексте и переписываются **набело** под 11 уровней (старый плагин — только
справка, его текст не копируется).

## Состояние

Наполнено (компонент 01 — готово): `framework.yaml` — каркас 11 уровней как данные (+ `framework.py`
— валидация и генерация `framework.md`, детерминированная проекция); `glossary.yaml` — глоссарий;
`passport.template.md` — шаблон паспорта проекта; `principles.default.md` — правила по умолчанию;
`theme.py` + `themes/` — темы оформления («Первый Бит»); `teamwork.md` — регламент командной работы;
`guide/` — «Руководство пользователя» (asset продукта, спека 06-f): `руководство.md` — исходник,
`build.py` — сборка в `.docx` (копируется в корень проекта на онбординге).
