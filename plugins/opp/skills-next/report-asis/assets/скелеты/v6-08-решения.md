# Скелет раздела: Целевые решения (тело, раздел 08)

Инструкция агенту-редактору: собрать нарратив о целевых решениях ИЗ памяти. Против каждой группы
проблем — совокупный замысел устранения по триаде (методология/организация/технология). Решения
группируются и нумеруются так же, как проблемы.

## Что читать из памяти
- **SOL** (решения): `название`, `формулировка`, `тип решения` (методологическое/организационное/
  технологическое), `уровень устранения`, `устраняемая проблема`, `закрываемое требование`,
  `статус проработки`, `обоснование выбора`.
- **DEC** (заключения-развилки): `предмет выбора`, `критерии сравнения`, `обоснование`,
  `компромиссы / цена выбора`, `отклонённые варианты — почему`, `статус решения` — для узлового
  решения (развилки платформы/ядра); вердикт не выносить, если статус не «согласовано».
- Порядок групп — тот же, что в разделе «Проблемы» (одна группа проблем → одна группа решений).

## Структура и порядок изложения
1. Вводный абзац (без подзаголовка): триада типов решений; сквозной принцип очерёдности («сначала
   порядок — методология и НСИ, затем автоматизация»); счёт решений по типам.
2. При наличии развилки — врезка `> **Узловое решение.** …` по DEC: в чём выбор, от чего зависит,
   рекомендованный шаг; вердикт оставить за заказчиком, если DEC не «согласовано».
3. По каждой группе — `### Решения для группы: <название группы>`:
   - 1 абзац: корень группы и логика устранения (что первично — методология/организация/технология).
   - **таблица**: `| Код | Решение | Тип | Содержание | Ожидаемый вклад |`; в каждой группе решения
     упорядочены по типу (методологические → организационные → технологические).
   - врезка `> **Ожидаемый эффект.** …` — какой ОБЭ группы снимается.

## Альтитуда
Тело — замысел устранения по группам. Не расписывать проектную реализацию (это следующий такт,
целевая архитектура/дорожная карта — отдельные документы). Содержание решения — в одну ячейку.

## Правила формулировок
- Экспертный русский решений; тип решения давать курсивом (`*методологическое*`). Коды решений —
  читаемые шифры проекта (например «Р01»), не внутренние ключи памяти; `PRB-…`, `TB-…`, `Ур.N` в
  текст не выносить.
- Обезличенные обороты: «корень группы организационный — у справочников нет владельца-источника»,
  «неупорядоченное автоматизировать нельзя, поэтому формализация правил предшествует переносу в
  систему».

## Честность
Только решения из памяти со своим статусом проработки. «Согласовано» не приписывать тому, что
«предложено»/«проработано». По развилке — приводить обе стороны и цену выбора, а не подсовывать
готовый вердикт, если его нет в DEC.

## Целевой объём
Пропорционально числу групп: ~0.5 страницы на группу (подводка + таблица + эффект).
