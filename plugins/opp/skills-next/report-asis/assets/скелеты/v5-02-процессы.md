# Скелет приложения Б: Детальная карта процессов и операций

Инструкция агенту-редактору: собрать полную пооперационную карту процессов ИЗ памяти. Приложение —
опора для построчной проверки утверждений тела (раздел «Бизнес-процессы»); разворачивается вся
декомпозиция функция → процесс → подпроцесс → операция.

## Что читать из памяти
- **PRC** (процессы) — все поля: `Название`, `Уровень декомпозиции`, `Родитель`, `Владелец`,
  `Вход`, `Действие`, `Выход`, `Система`, `Порядок в потоке`, `Точка решения`,
  `Полнота описания`, `Статус доказанности`, `локатор в источнике`.
- **PSK** (скелет процессов): контуры, подтверждённые только на уровне скелета.
- **INT** (Ур.9): межсистемные потоки, привязанные к операциям, — для отметки интеграций в шагах.

## Структура и порядок изложения
1. `### Функциональная модель (обзор)` — определения уровней (функция/процесс/подпроцесс/операция);
   счёт по каждому уровню; правило: процесс без подпроцессов даёт операции прямо в своём разделе.
   **Таблица функций** `| Код | Функция | Владелец | Краткое описание | Системы |`.
2. `**Полная иерархия процессов.**` — **таблица-дерево** `| Код | Наименование |` со всеми узлами,
   отступ-дефисами по уровню вложенности (родитель по полю `Родитель`).
3. По каждому процессу/подпроцессу — заголовок соответствующего уровня (`####`/`#####`) и **таблица
   операций** по порядку потока: `| № | Операция | Владелец | Вход | Действие | Выход | Система |`,
   со стартовым и финальным событиями. Где подтверждён только скелет — строка
   `*Топология требует подтверждения заказчиком.*` вместо таблицы шагов.

## Альтитуда
Приложение — самый детальный слой отчёта. Здесь и только здесь приводятся построчные таблицы
операций. Обобщающих выводов не делать — они в теле.

## Правила формулировок
- Экспертный процессный русский; шаги — глаголами действия; событие старта/финала — явно.
- Шифры функций/процессов — читаемые; внутренние ключи узлов памяти и `Ур.N` в текст не выносить.

## Честность
Приводить только подтверждённые шаги. Неполный/скелетный процесс — прямая пометка «последовательность
не выверена, вынесена в открытые вопросы». Атрибут шага (владелец/вход/действие/выход/система)
печатать только если поле заполнено; пустое — прочерк, не догадка.

## Целевой объём
Не ограничен — приложение полное: все функции, процессы, подпроцессы и операции памяти.
