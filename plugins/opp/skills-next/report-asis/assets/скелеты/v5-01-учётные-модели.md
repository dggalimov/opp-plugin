# Скелет приложения А: Паспорта учётных моделей

Инструкция агенту-редактору: собрать детальные паспорта учётных моделей ИЗ памяти. Приложение — опора
для построчной проверки утверждений тела (раздел «Учётная модель»); здесь допустима и нужна
максимальная детализация.

## Что читать из памяти
- **AM** (учётные модели) — все поля: `название`, `класс модели`, `назначение модели`,
  `краткое описание`, `ключевые измерения`, `ключевые факты модели`, `где живёт (система)`,
  `методолог / владелец`, `статус доказанности`, `источник`, `локатор в источнике`.
- **BR** (Ур.3): бизнес-правила, привязанные к модели, — их суть и связанная проблема.
- **NSI** (Ур.4): справочники/аналитики, питающие модель.
- **REP** (Ур.1): отчёты/метрики на выходе модели.
- **SYS** (Ур.7): системы, где живёт модель. **PRC** (Ур.5): связанные процессы.

## Структура и порядок изложения
1. Вводный абзац: что такое учётная модель (методология: измерения, факты, правила, справочники).
2. `### Сводная таблица учётных моделей` — одна строка на модель: `| Код | Учётная модель | Краткое
   описание | Бизнес-правила | Синтетическая структура | Аналитики | Отчёты | Системы | Процессы |`.
   Значения сжаты до ключевых имён; полное раскрытие — в паспортах ниже.
3. По каждой модели — `### <Код>. <Название>` — паспорт по единому формату:
   `**Код учётной модели.**`, `**Учётная модель.**`, `**Краткое описание.**`, `**Бизнес-правила.**`
   (таблицей `| Правило | Суть | Проблема |`), `**Синтетическая структура.**`, `**Аналитики.**`,
   `**Отчётность.**`, `**Системы.**`, `**Связанные процессы.**`. Родственные контуры можно свести в
   один паспорт как грани одной модели.

## Альтитуда
Приложение — детальный слой. Здесь разворачиваются все измерения, факты, правила и аналитики,
опущенные в теле. Каждый паспорт самодостаточен.

## Правила формулировок
- Экспертный учётно-методологический русский; термины — по глоссарию заказчика (GLO).
- Коды моделей — читаемые шифры проекта; внутренние ключи памяти и `Ур.N` в текст не выносить.
- Коды процессов в паспорте — читаемыми шифрами функций (раскрытие — Приложение Б).

## Честность
Паспорт содержит только заполненные поля модели. Пустой пункт формата помечать «не обследовано,
вынесено в открытые вопросы», а не достраивать. Правило приводить в паспорте только если оно есть
в BR; связанную проблему — только по фактической ссылке.

## Целевой объём
Не ограничен сверху — приложение полное. Ориентир: сводная таблица + паспорт (~0.5–1 страница) на
каждую модель памяти.
