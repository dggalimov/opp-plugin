# Скелет раздела: Учётная модель предприятия (AS-IS) (тело, раздел 03)

Инструкция агенту-редактору: собрать нарратив об учётной архитектуре «как есть» ИЗ памяти. Раздел
тела даёт укрупнённую картину учётных моделей; детальные паспорта каждой модели — в Приложении А.

## Что читать из памяти
- **AM** (Ур.2, учётные модели): `название`, `класс модели`, `назначение модели`, `ключевые
  измерения`, `ключевые факты модели`, `где живёт (система)`, `методолог / владелец`,
  `статус доказанности`. Это ядро раздела.
- **BR** (Ур.3, бизнес-правила): ключевые алгоритмы/политики, которыми модель считается — для
  описания природы и болезней моделей.
- **NSI** (Ур.4): справочники и мастер-системы, от которых модели наследуют аналитику — для узла
  «двойное кодирование / теневые звенья», если он есть в фактах.
- **REP** (Ур.1): какие метрики/отчёты порождают модели — для связи «модель → метрика».
- **PRB**/**EFF**: проблемы, локализованные на учётных моделях/правилах/НСИ — для блока узких мест.

## Структура и порядок изложения
1. `### Обзор учётной архитектуры` — 1–2 абзаца: что понимается под учётной моделью (методология:
   измерения, факты, правила, справочники — не отчёт и не система); сколько моделей описывает
   контур; общие сквозные черты (например, параллельные кодировки, теневые звенья) — если они есть в
   фактах. Завершить врезкой `> **Ключевой вывод.** …`.
2. `### Ключевые учётные модели` — 1 вводный абзац + **таблица**: `| Учётная модель | Назначение |
   Где реализована | Ключевые узкие места |`. Одна строка на модель; полные паспорта — Приложение А.
3. `### Узкие места учётной архитектуры` — врезка `> **Узкое место.** …` с 2–3 корневыми дефектами,
   проходящими сквозь модели, каждый — абзацем; связать с группами раздела «Проблемы».

## Альтитуда
Тело — обобщение по моделям и корневым дефектам. Полные паспорта (бизнес-правила, синтетическая
структура, аналитики, отчёты, системы, связанные процессы) НЕ разворачивать — они в Приложении А.

## Правила формулировок
- Экспертный учётно-методологический русский; термины учёта — по глоссарию заказчика (GLO), если он
  ведётся. Без кодов памяти и без `Ур.N`.
- Обезличенные обороты: «Отчётность — верхний слой; под ним лежат учётные модели, задающие разрезы и
  факты; ещё ниже — бизнес-правила; в основании — справочники (НСИ)», «достоверность держится на
  ручном труде, а не на системе».

## Честность
Только модели и дефекты, зафиксированные в памяти. Модель, по которой известно лишь название без
измерений/фактов, помечать состоянием «обследована частично, детали вынесены в открытые вопросы», а
не дописывать правдоподобные измерения. Узкое место называть, только если оно опирается на PRB/EFF.

## Целевой объём
1–2 страницы (≈ 700–1100 слов), без учёта таблицы моделей.
