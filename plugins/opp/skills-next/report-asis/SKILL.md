# Навык: Отчёт об обследовании AS-IS

> Навык ПРОДУКТА. Отчёт — проекция зафиксированных фактов: никакой прозы от себя,
> никаких правок документа руками. Правда живёт в памяти; документ пересобирается.

## Предусловия (без них отчёт не выдаётся)

1. `"${CLAUDE_PLUGIN_ROOT}/opp" lint "<проект>"` — 0 ERROR (замкнутость, гейт ОБЭ, трассировка).
2. `"${CLAUDE_PLUGIN_ROOT}/opp" prove --into "<проект>"` — 0 дефектов доказательств (величины воспроизводимы
   или честно пусты).
3. `"${CLAUDE_PLUGIN_ROOT}/opp" requirements --into "<проект>"` — 0 непокрытых проблем и требований без проблемы
   (или расхождения объяснены в обзоре).
Дефекты есть → сначала чинится память (навыком синтеза/каналов), потом отчёт.

## Сборка

```bash
"${CLAUDE_PLUGIN_ROOT}/opp" render asis --into "<проект>" --docx
```

Выход: `<проект>/Документы/asis.md` (канонический) + `asis.docx` (представление с темой).

## Сверка полноты (перед выдачей)

- Каждая таблица профиля с непустой памятью попала в документ секцией (пустые секции
  скрыты по конфигу — это норма; непустая таблица, отсутствующая в конфиге, — дефект
  конфига, чинить `documents/asis/config.yaml`).
- Счётчики шапки-паспорта сходятся со счётом строк памяти.
- Секция «Главное» держит только проблемы со статусом выше «гипотезы».
- Величины ущерба: пустые значения — это честное «ожидает данных заказчика»
  (см. открытые вопросы), НЕ дефект. Не заполнять руками.

## Выдача и правки

Документ уходит заказчику/владельцу на просмотр. Замечания по СОСТАВУ/ВИДУ — правка
`documents/asis/config.yaml` (+ golden пересчитать) и пересборка. Замечания по ФАКТАМ —
через память: правки протоколов каналов (`"${CLAUDE_PLUGIN_ROOT}/opp" split` протокола) или навыки; затем
пересборка отчёта. Прямое редактирование asis.md/docx запрещено — потеряется при
следующей сборке, и это правильно.

## Границы

НЕ меняет память; НЕ пишет прозу-резюме от себя; НЕ выдаёт отчёт при красных
предусловиях; PDF и брендированные шаблоны — вне финала.
