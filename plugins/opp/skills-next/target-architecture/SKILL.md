# Навык: Целевая архитектура TO-BE

> Навык ПРОДУКТА. TO-BE — не мечта: каждый целевой образ обоснован решением, каждое
> решение приземлено на уровень каркаса. Симметрия: у AS-IS-картины уровня есть целевой
> образ или явное «без изменений».

## Несущие правила

1. **Обоснованность.** Каждый элемент TLM ссылается на решение (SOL); целевых образов
   «потому что красиво» нет. `"${CLAUDE_PLUGIN_ROOT}/opp" tobe` проверит.
2. **Симметрия каркасу.** Пройди все 11 уровней: где AS-IS несёт проблемы выше «гипотезы» —
   целевой образ обязателен; где изменений нет — явная строка «без изменений» с причиной
   (белых дыр молча не оставляем).
3. **Мастер-системы — явно** (методика р.5): целевая картина НСИ называет мастер-систему
   каждого справочника; целевые интеграции — направление и владельца потока.
4. **Не перерешивай решения.** TLM раскладывает ПРИНЯТЫЕ SOL по уровням; новое решение
   понадобилось — вернись в навык решений, не чекань его тут.
5. **Дедуп и слаги** — по словарю; «уже_в_памяти» прежде чеканки.

## Как устроено (важно)

Целевая модель по уровням (TLM) — таблица-ПРЕДСТАВЛЕНИЕ: она вычисляется из решений
(`SOL.уровень устранения` + описание образа), руками не заполняется (C3 — рельсы отклонят).
Твоя работа — чтобы у КАЖДОГО уровня каркаса целевая картина выводилась из SOL: либо
решение с уровнем устранения и целевым образом, либо явная пометка «без изменений»
с причиной в строке решения/примечании.

## Процедура

1. Вход: `"${CLAUDE_PLUGIN_ROOT}/opp" solutions --into "<проект>"` — чисто. Прочитай SOL, DEC, картины уровней
   AS-IS (extract даёт предшественников), `reference/framework.md` (уровни).
2. Пройди уровни 1–11 по AS-IS: где есть проблемы выше «гипотезы» — у решений должен быть
   заполнен «уровень устранения» и целевой образ; дополни/уточни строки SOL слиянием
   (write_rows); уровень «без изменений» — явная пометка.
3. Самопроверка: `"${CLAUDE_PLUGIN_ROOT}/opp" tobe --into "<проект>"` — 0 дефектов (белых пятен TO-BE нет или
   объяснены); `"${CLAUDE_PLUGIN_ROOT}/opp" lint` — 0 ERROR.
4. Обнови `обзор.md`: целевая картина по уровням, что «без изменений», развилки.

## Запись строк — только через рельсы (write_rows, как во всех навыках).

## Границы

НЕ выбирает продукты (решения); НЕ строит волны (дорожная карта); НЕ проектирует модель
данных до полей (TDM вне финала); НЕ рисует BPMN.
