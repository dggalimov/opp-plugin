# Навык: Собрать базу знаний

Помогает сотруднику по общим правилам собрать ещё одну БЗ — это элемент масштабирования продукта.

## 1. Определить тип (технологию)
По характеру источника выбери тип (см. `${CLAUDE_PLUGIN_ROOT}/knowledge/build-types.yaml`):
- **индексная** — метаданные/XML-выгрузка конфигурации 1С (точный поиск/diff);
- **семантическая (RAG)** — тексты: ИТС, CIMA, публичные доки, архив;
- **плоская** — маленький текст, всегда в контексте;
- **шаблоны** — образцы (эталоны);
- **навигатор** — верхнеуровневая идентификация продукта.

## 2. Что попросить у пользователя
По выбранному типу примени инструкцию из `build-types.yaml` (что дать/сказать/попросить). Кратко: для
индексной — XML-выгрузку типовой; для семантической — файлы/ссылку на тексты; для плоской/шаблонов —
готовые файлы; для навигатора — краткие описания продуктов.

## 3. Собрать артефакт
- **Плоская / шаблоны / навигатор** — сложить файлы в `${CLAUDE_PLUGIN_ROOT}/knowledge/bases/<id>/`
  (читаются напрямую единым интерфейсом `knowledge/query.py`).
- **Индексная / семантическая** — тяжёлые бэкенды обслуживает **MCP `opp-knowledge`**
  (разрабатывается параллельно; см. `knowledge/mcp-development-plan.md`). Сборку индекса/векторов
  выполняет конвейер MCP; здесь — подготовить источник и зарегистрировать БЗ.

## 4. Зарегистрировать и привязать
- Обнови строку элемента БЗ в **`plan/03.yaml`** (единый источник): `статус`, `даёт` (роль · домен ·
  привязка), `потребляет` (источник · технология), `где`. Готовую БЗ ставь в `проверка` — в `готово`
  переводит OPP 00.
- При необходимости добавь привязку «БЗ → процедура заполнения реестра» в `knowledge/bindings.yaml`
  (это подсказка, не факт).

## 5. Проверить
`"${CLAUDE_PLUGIN_ROOT}/opp" verify` (таблица валидна) → при наличии окна `"${CLAUDE_PLUGIN_ROOT}/opp" ui` разделы «Базы знаний»/«План» показывают
БЗ со статусом.

## Контракт
Вход: источник знаний + тип. Выход: собранная/зарегистрированная БЗ, доступная по `id` через единый
интерфейс. Тяжёлую индексацию/векторизацию выполняет MCP `opp-knowledge`.
