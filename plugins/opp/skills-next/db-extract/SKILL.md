# Навык: Срез данных и журнала из базы 1С

Снять **осмысленный** (не дамп) срез данных и журнала по профилю конфигурации.

## 1. Определить конфигурацию и профиль
По `SRC-CFG` (срез конфигурации) и навигатору определи продукт. Возьми **профиль сбора**:
`${CLAUDE_PLUGIN_ROOT}/capabilities/extract/profiles/<код>.yaml` (или профиль проекта, если правлен).
Профиль задаёт, что собирать и на чём фокус (БП → регистры бухгалтерии; УТ → регистры накопления).

## 2. Путь извлечения
- **Доступ к базе есть** → через `mcp-1c`: `execute_query` (статистика по профилю), `get_event_log`
  (журнал). Запросы **только на чтение**, лёгкие, с оглядкой на нагрузку.
- **Доступа/MCP нет** → попроси выгрузки (наши 1С-обработки / Excel / XML / журнал файлом); прими их
  через `"${CLAUDE_PLUGIN_ROOT}/opp" ingest`, затем сведи в узлы ниже.

## 3. Что собрать (по профилю)
Срез данных: справочники (состав/число/заполненность/дубли), документы (статистика), регистры
бухгалтерии/накопления/сведений (число записей, обороты, используемые разрезы), объёмы, **справочник
дополнительных обработок и отчётов** (кустарные доработки вне конфигурации). Журнал: реальные активные
пользователи, реально используемые объекты, активность по времени.

## 4. Сохранить как SRC-DAT и SRC-LOG (с ключом базы)
Узлы кладём в `<проект>/Источники/База данных/` с тем же `ключ_базы`, что у `SRC-CFG` этой базы —
тогда синтез смотрит конфигурацию + данные + журнал совместно (+ доменная БЗ).
```
---
id: SRC-DAT-001
kind: срез данных
title: <база> — данные
origin: mcp-1c | выгрузка
readable: true
ключ_базы: БАЗА-001
used_by: []
---
<агрегаты по профилю, таблицами>

## Фокус
Ур.2/4 (НСИ, учётные модели), Ур.5 (процессы по статистике), расхождения данных и правил.
```
Аналогично `SRC-LOG`. Проверь: `"${CLAUDE_PLUGIN_ROOT}/opp" verify`.

## Контракт
Вход: SRC-CFG + профиль + доступ/выгрузки. Выход: SRC-DAT и SRC-LOG с ключом базы. Сбор осмысленный,
агрегированный, без избыточного дампа; контент полон для синтеза.
