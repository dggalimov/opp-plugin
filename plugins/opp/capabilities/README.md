# capabilities — операции

Атомарные операции продукта с машинным контрактом (вход → выход):

- **Извлечение (E)** — превратить любой вход (документ, видео, аудио, выгрузку из базы) в источник
  единого вида.
- **Преобразования (T)** — операции над проектной памятью: разложить по таблицам, сверить, доказать,
  собрать требования и решения, спроектировать целевую модель, проверить замкнутость.
- **Сборка документов (L)** — детерминированно собрать готовый документ из таблиц фактов.

## Состояние

Наполнено: `deploy.py` — идемпотентный разворот рабочей папки (L-заготовка, компонент 01);
`extract/` — конвейер источников-«бронзы» (E, компонент 02 — готово): ingest, конвертеры форматов,
встречи (Newton), срез конфигурации, профили сбора, архитектура источников.
T-преобразования физически живут рядом с контрактом схемы: `schema/extract.py` (сырьё → строки),
`schema/instructions.py` (инструкции), `synthesis/triangulate.py` (сверка источников, вход синтеза).
