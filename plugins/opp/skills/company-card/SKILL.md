# Навык: Карточка компании (публичные источники)

Собрать публичные факты о компании заказчика и положить в бронзу как `SRC-COM`.

## 1. Вход
Паспорт проекта (заказчик, отрасль, юрлица) + материалы. Если данных мало — спроси ИНН/название.

## 2. Ресёрч
По открытым источникам собери (через веб-поиск/ресёрч): группа компаний и **юрлица**; **обороты и
фин. показатели**; **виды деятельности**; **активы**; признаки вхождения в **холдинги**;
**собственники и топ-менеджмент**; масштаб, география; всё, что полезно для аудита.

## 3. Доказуемость
Это **публичные данные** → статус **гипотеза**, провенанс — ссылки на источники. Не выдавать за
подтверждённый факт; каждый блок — со ссылкой на источник.

## 4. Сохранить как SRC-COM
Создай узел `<проект>/Проектная память/Источники/Документы/SRC-COM-NNN.md` по образцу узла источника:
```
---
id: SRC-COM-001
kind: карточка компании
title: <Заказчик> — публичные данные
origin: публичные источники
readable: true
ключ_базы: null
used_by: []
---
<факты по блокам, каждый со ссылкой>

## Фокус
Ур.1 (масштаб/отрасль для метрик), Ур.6 (оргструктура/собственники), периметр.
```
Проверь: `"${CLAUDE_PLUGIN_ROOT}/opp" verify`. В окне раздел «Источники» покажет узел в реестре.

## 5. Показать пользователю
Сохранение узла — не конец: выведи карточку **сжатым текстом прямо в чат** пользователю на
сверку — не только узел в реестре. Структурируй блоками: **юрлица**, **обороты**,
**деятельность**, **холдинг**, **руководство** (показывай только те блоки, где реально есть
данные; пустой блок не показывай). У каждого факта — краткое указание источника (домен/название
ресурса). Попроси подтвердить или поправить; правки пользователя вноси в `SRC-COM`.

## Контракт
Вход: паспорт/материалы. Выход: `SRC-COM` с публичными фактами и ссылками (статус — гипотеза)
+ карточка сжатым текстом в чате пользователю на сверку (не только запись в реестре).
