# OPP — установка плагина

Это маркетплейс Claude Code с плагином `opp` — ведение проекта внедрения 1С/ERP одним
архитектором с ИИ.

## Установка

В интерактивном `claude`:
```
/plugin marketplace add <репозиторий-или-путь>
/plugin install opp@opp
```
`<репозиторий-или-путь>` — адрес опубликованного репозитория (напр. `owner/opp-plugin`) или
локальный путь к распакованной папке маркетплейса.

Первый запуск любой команды (напр. `/opp:new-project`) сам поднимет python-окружение плагина
(`.venv` рядом с плагином) и поставит основные зависимости — увидите сообщение
«OPP: первый запуск, готовлю окружение…». Дальше — тихо.

## Требования

- **python3 ≥ 3.12** — обязательно, без него плагин не запустится.
- `.xlsx`/`.xlsm`/`.docx` читаются без доп. установок (python-native: openpyxl, python-docx —
  ставятся вместе с плагином).
- Дальше — опционально, по фичам (плагин честно сообщает, если чего-то не хватает, и не падает):
  - **soffice (LibreOffice)** — старые/иные офисные форматы заказчика (`.doc`, `.ppt`, `.pptx`,
    `.xls`, `.xlsb`, `.ods`).
  - **tesseract + pdftotext + pdftoppm** — PDF-сканы и картинки (OCR): `pdftotext` для текстового
    слоя PDF, `pdftoppm` + `tesseract` для сканов без текстового слоя и отдельных изображений.
  - **ffmpeg + newton** — транскрибация встреч из аудио/видео.
  - **java** — чтение `.mpp` (Microsoft Project).
  - **gh (GitHub CLI)** — шаг подключения проекта к GitHub в онбординге.

## Команды

- `/opp:new-project` — онбординг нового проекта: структура, паспорт, карточка компании,
  оформление, гайд.
- `/opp:analyze-materials` — анализ входящих материалов проекта (документы, выгрузки).
- `/opp:plan-meetings` — планирование цикла встреч по проекту.
- `/opp:analyze-meeting` — разбор проведённой встречи (протокол, факты, вопросы).
- `/opp:synthesis` — синтез собранных фактов в требования и гипотезы.

## Обновление

Обновление каталога само по себе НЕ обновляет установленный плагин — нужна переустановка:
```
/plugin marketplace update opp
/plugin uninstall opp@opp
/plugin install opp@opp
```
Проекты и их данные при переустановке не трогаются — плагин живёт отдельно от рабочих папок.

Чтобы новые версии подхватывались сами при старте `claude`: `/plugin` → вкладка
**Marketplaces** → включить автообновление для `opp`. Плагин ставится на пользователя
(один на машину): обновили один раз — действует во всех проектах и сессиях этой машины;
на другой машине повторите те же команды.
