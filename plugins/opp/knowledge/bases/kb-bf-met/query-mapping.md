# kb-bf-met — маппинг БИТ.Финанс-запросов

Назначение: привести запросы аудитора (живой язык) к **терминологии руководства БИТ.Финанс**, чтобы RAG
находил релевантные пассажи. Слева — тема/как спрашивают; справа — термины-якоря для запроса к RAG.
Термины выверены по руководству (проверка на rag.1bitai.ru, см. `test-questions.yaml`).

## По подсистемам

**Бюджетирование**
- темы: планы, бюджеты, БДР/БДДС, план-факт, аналитики бюджета
- якоря: «Статьи оборотов», «Сценарии бюджетирования», «ЦФО», «Проекты», «Номенклатурные группы», «Планирование зависимых статей оборотов», «Соответствие статей оборотов и статей регламентированного учёта», «Экземпляр бюджета»

**Казначейство**
- темы: оплаты, заявки, платёжный календарь, реестр, поступления
- якоря: «Заявка на расходование денежных средств», «Реестр платежей», «Планируемое поступление денежных средств», отчёт «Платёжный календарь», «План-фактный анализ по прогнозам платежей»

**Управление договорами**
- темы: договоры, лимиты, доп. условия
- якоря: «Дополнительные условия по договору», контроль суммы договора, лимиты по договору

**Управленческий учёт / МСФО**
- темы: параллельный учёт, трансформация, IFRS
- якоря: «БИТ.ФИНАНС:Управленческий учёт», «Механизм трансляции», параллельный учёт (ОС, НМА, аренда, финансовые инструменты, отложенные налоги), закрытие счетов, пакеты отчётности по МСФО

**Консолидация**
- темы: группа компаний, внутригрупповые обороты
- якоря: «Сверка внутригрупповых оборотов» (версия Холдинг), консолидированная отчётность

**Общее/UI (для идентификации, не для методики)**
- якоря: «Настройки программы (БИТ)», «Настройки (БИТ)», «Быстрый старт», «БИТ: Система лицензирования»

## Правила формирования запроса к RAG
1. Брать **точные названия объектов** руководства (документ/отчёт/справочник/регистр) — у них высокий скор (напр. «Заявка на расходование денежных средств» → score 1.0).
2. Если вопрос общий («как устроено бюджетирование») — добавлять 2–3 термина-якоря из таблицы.
3. `top_k=5` по умолчанию; для узких объектов хватает `top_k=3`.
4. Не подмешивать термины ИТС/типовой конфигурации — это отдельная база (`kb-<base>-met`). БИТ.Финанс ≠ типовой казначейство/бюджетирование.
5. Ответ всегда с провенансом (файл · чанк); нет релевантного — «не найдено», не выдумывать.
