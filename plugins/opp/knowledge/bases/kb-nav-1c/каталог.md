# Навигатор по продуктам 1С (kb-nav-1c)

Назначение: по **косвенным признакам** (что видно на экране, в документах, в речи заказчика) опознать
конфигурацию + надстройки и направить к нужным доменным базам знаний. База **высокоуровневая**: только
признаки опознания, без how-to и методологии.

Принцип доказуемости: каждый признак привязан к источнику (см. строку «Источник» в карточке) и опирается
на **комбинации**, а не на одиночные общие слова (см. §3). Опознание — гипотеза, которую подтверждают.

## §1. Определитель (быстрый ключ — от грубого к точному)

1. **Сторонняя платформа, не 1С?** Интерфейс не-1С (EN/ES/PT, веб), вендор PIX Robotics, имена PIX
   (RPA/BI/Process Mining/Operator/Flexform), PIX Master/Studio/Robot, «цифровые сотрудники» → **PIX**
   (внешний; 1С-ветку дальше не идём).
2. **Инфраструктурный слой обмена?** Каналы / сообщения / маршруты / брокер / оркестрация, мониторинг
   доставки → **1С:Шина**.
3. **Слой управления содержанием справочников?** Заявка на изменение НСИ → эксперт → эталонный объект →
   репликация; дедубликация, классификация, выравнивание атрибутов → **1С:MDM**.
4. **Аналитика/дашборды как сам продукт?** Поверх одной базы 1С без ETL/коннекторов → **1С:Аналитика**;
   внешняя платформа с коннекторами к разным системам → **PIX BI**.
5. **Адресный склад?** Ячейки/зоны хранения, ТСД, приёмка→размещение→отбор→отгрузка → **1С:WMS**.
6. **Перевозки?** Рейсы, заявки на перевозку, привлечённый транспорт, маршрутизация/тарификация, последняя
   миля → **1С:TMS**.
7. **Строительная первичка КС-2/КС-3/М-29/КС-6а, объекты строительства, эскроу, ДДУ, ПБУ 2/2008?**
   → надстройка **БИТ.Строительство** (поверх базовой, обычно 1С:Бухгалтерия КОРП).
8. **Казначейско-бюджетные объекты СВЕРХ типового** (Заявка на расход ДС + Реестр платежей + Платёжный
   календарь + Статьи оборотов/ЦФО/Сценарии) и **UI-маркеры «БИТ»** → надстройка **БИТ.Финанс**
   (версия Холдинг — если есть «Сверка ВГО»).
9. **Только кадры и зарплата отдельной базой** (штатное расписание, табель, НДФЛ/взносы, перс.учёт,
   статотчётность), без бухучёта и производства → **1С:ЗУП** (КОРП — если кадровый ЭДО/подбор/охрана труда).
10. **Есть ли консолидация группы / элиминация ВГО / трансформация МСФО как контур?**
    → **Да** — ветка группы (11–12); **Нет** — ветка одиночной базы (13–16).
11. *(группа)* Только параллельный учёт/трансформация по МСФО для одной организации, без консолидации
    группы → **1С:Бухгалтерия КОРП МСФО**. Полное управление группой (консолидация + казначейство +
    мастер-данные + ССП) → шаг 12.
12. *(группа)* Ведёт первичный оперативный учёт (производство/склад/продажи) в той же базе?
    **Да, встроено в ERP** → **1С:ERP.УХ**; **Нет, сборщик поверх баз** (сбор отчётности от организаций
    группы) → **1С:УХ**.
13. *(одиночная)* **Есть ли регламентированный учёт в самой базе?** Нет → 14–15; Да → 16.
14. *(нет полноценного регучёта)* Весь контур (продажи/услуги/склад/CRM) + упрощённое производство +
    зарплата + упрощённая отчётность в одной лёгкой базе, микро/малый бизнес (Заказ-наряд) → **1С:УНФ**.
15. *(нет регучёта)* Только торгово-складской контур + CRM + казначейство; данные уходят в 1С:Бухгалтерию;
    управляет 1С:Розница; редакция 11 → **1С:УТ**.
16. *(есть регучёт)* **Есть ли производство и какое?**
    - Нет / только базовый учёт, ядро = регламентированная отчётность (ОСВ, книги, декларации, ЕНС,
      1С-Отчётность) → **1С:Бухгалтерия** (КОРП — обособленные подразделения/многофирменность).
    - Облегчённое производство + регучёт + зарплата + торговля в одной базе, без контура ERP/WMS/МСФО,
      МСБ-позиционирование (ред. 2, версия 2.5.x) → **1С:КА**.
    - Полное производство + регучёт в одной базе → 17.
17. *(полное производство + регучёт)* **Платформа?** 8.2, ред. 1.3, снято с развития, миграция В ERP →
    **1С:УПП**; 8.3, «Такси», ред. 2, переход С УПП и миграция с зарубежных ERP, есть WE/WE ENG → **1С:ERP**.

## §2. Карточки-сигнатуры

Легенда: **тип** — база / надстройка / внешний. Каждая карточка привязана к источнику (строка «Источник»).

### A. Комплексные ERP-контуры

**1С:ERP Управление предприятием** · `key: erp` · база
- Подсистемы: Управление производством, Регламентированный учёт, Международный финансовый учёт,
  Управление финансами и бюджетирование, Казначейство, Затраты/себестоимость, Продажи/Закупки/Склад,
  Управление персоналом и зарплата, Организация ремонтов.
- Маркеры: «1С:ERP Управление предприятием 2», версия 2.5.x; редакции «1С:ERP WE»/«WE ENG»; платформа 8.3
  (интерфейс «Такси»); блок «Переход с 1С:УПП»; «Миграция с зарубежных ERP-систем».
- Различающая сигнатура: **производство (полный цикл) + регучёт + МСФО в одной базе на 8.3, БЕЗ контура
  консолидации группы** (с консолидацией/элиминацией ВГО → это ERP.УХ).
- Негатив: не УПП (8.2, снят с развития); не УТ (нет регучёта); не КА (там производство облегчённое).
- Маршрут: `erp` → kb-erp-str/met (с БИТ.Финанс → kb-erp-bf-*). Уровни: 1,2,3,5,8. Источник: v8.1c.ru/erp.

**1С:ERP. Управление холдингом (ERP.УХ)** · `key: erpuh` · база
- Класс ERP+ : ERP-учётное ядро + контур холдинга в едином продукте на платформе 8.3.
- Функциональность: Корпоративное управление (консолидация финансовой отчётности), Финансы и персонал,
  Продажи и клиенты, Закупки и производство, Коммуникации между системами и пользователями.
- Для кого: холдинги/группы компаний, крупные предприятия (единое решение в управляющей компании и «дочках»).
- Различающая сигнатура: **первичный оперативный учёт ERP (производство/склад/продажи) ВМЕСТЕ с
  корпоративным управлением/консолидацией группы в ОДНОМ продукте** (учётное ядро встроено — не сборщик).
- Негатив: не УХ (тот — сборщик поверх чужих баз); не чистый ERP (есть контур холдинга).
- Маршрут: `erpuh` → kb-erp-uh-str/met. Уровни: 1,2,4,3,6. Источник: v8.1c.ru/cpm-erp.

**1С:Комплексная автоматизация (КА)** · `key: ka` · база
- Подсистемы: Бюджетирование, Казначейство, CRM, Продажи, Склад/Закупки, Затраты/себестоимость,
  Управление персоналом и зарплата, Бухгалтерский и налоговый учёт; производство — облегчённое («учёт в производстве»).
- Маркеры: «Комплексная автоматизация, редакция 2» / «1С:КА 2», версия 2.5.x; платформа 8.3.
- Различающая сигнатура: **полный учётный контур (регучёт+зарплата+торговля) + ОБЛЕГЧЁННОЕ производство в
  одной базе, БЕЗ контура ERP, БЕЗ WMS, БЕЗ МСФО/консолидации** — средняя ступень между УТ и ERP.
- Маршрут: `ka` → kb-ka-str/met (с БИТ.Финанс → kb-ka-bf-*). Уровни: 1,2,5,8. Источник: v8.1c.ru/ka.

**1С:УПП (Управление производственным предприятием)** · `key: upp` · база (legacy)
- Состав близок к ERP: производство + регучёт + кадры/зарплата старого образца в одной базе.
- Маркеры (главный различитель): **платформа 8.2** (не 8.3), старый интерфейс без «Такси», редакция 1.3,
  выпуск 2004, выведен из прайс-листа 30.04.2018, **снят с развития**, миграция → 1С:ERP 2.
- Различающая сигнатура: производство+регучёт как у ERP, **НО устаревшая платформа/поколение и статус «не
  развивается»** (состав подсистем сам по себе почти совпадает с ERP).
- Маршрут: `upp` → kb-upp-str/met. Уровни: 7,8,11,1. Источник: v8.1c.ru/upp (условия сопровождения).

### B. Торговля и малый бизнес

**1С:Управление торговлей (УТ)** · `key: ut` · база
- Подсистемы: Продажи (опт/розница/кредит/предзаказ/комиссия), Склад, Закупки, CRM, Казначейство,
  Затраты, Планирование; виды торговли как набор.
- Маркеры: «Управление торговлей, редакция 11», версия 11.5.x; есть Базовая; управляет «1С:Розница 8»;
  **данные передаются в «1С:Бухгалтерию 8»** (нет своего регучёта).
- Различающая сигнатура: **оперативный торгово-складской контур + CRM + казначейство, БЕЗ регучёта (уходит
  в БП), БЕЗ зарплаты, БЕЗ производства**.
- Маршрут: `ut` → kb-ut-str/met (с БИТ.Финанс → kb-ut-bf-*). Уровни: 1,5,9,4. Источник: v8.1c.ru/trade.

**1С:Управление нашей фирмой (УНФ)** · `key: unf` · база
- Возможности: Продажи (опт/розница/онлайн), Услуги, Производство, Склад, Закупки, Ценообразование, CRM,
  Финансы, Зарплата и персонал, Отчётность и налоги, Маркировка.
- Объекты/маркеры: **«Заказ-наряд»** (услуги/работы), Заказ покупателя, упрощённый Заказ на производство;
  поддержка 54-ФЗ/онлайн-касс, ЕГАИС/ГИСМ; облако 1cFresh; «всё в одном» для микро/малого бизнеса.
- Различающая сигнатура: **весь оперативный контур + услуги + упрощённое производство + зарплата +
  упрощённая отчётность в одной лёгкой базе для малого бизнеса/ИП**. vs УТ — есть производство/услуги/
  зарплата; vs БП — нет полноценного регламентированного бухучёта; vs КА — заметно легче.
- Маршрут: `unf` → kb-unf-str/met. Уровни: 1,5,7. Источник: v8.1c.ru/small.biz.

### C. Регламентированный учёт и финансы

**1С:Бухгалтерия (ПРОФ/КОРП)** · `key: bp` · база
- Подсистемы: Денежные средства, Расчёты с контрагентами, ТМЦ, Торговля, Производство (базовое), ОС, НМА,
  Зарплата и кадры (упрощённо), Закрытие месяца, Регламентированная отчётность.
- Объекты/отчёты: Реализация (УПД), Счёт-фактура, Закрытие месяца; ОСВ, Книги покупок/продаж, декларации.
- Маркеры: редакция 3.0, 1С-Отчётность, ЕНС/ЕНП; **«Обособленные подразделения», многофирменность, доп.
  валюты → КОРП** (иначе ПРОФ).
- Различающая сигнатура: **ядро = регламентированный учёт + отчётность, при упрощённых торговле/производстве/
  зарплате и БЕЗ казначейства-бюджетирования**. Если рядом заявки+платёжный календарь+статьи оборотов → это
  надстройка БИТ.Финанс поверх БП.
- Маршрут: `bp` → kb-bp-str/met (+БИТ.Финанс → kb-bp-bf-*; +БИТ.Строительство+БИТ.Финанс → kb-bp-bs-bf-*).
  Уровни: 2,3,1,8. Источник: v8.1c.ru/buhv8.

**1С:Управление холдингом 8 (УХ, отдельное решение)** · `key: uh` · база
- Функциональность: Бизнес-анализ и ССП, Регламентированный учёт, Корпоративная налоговая функция, Учёт и
  консолидированная отчётность по МСФО, Централизованное управление инвестпроектами, Интеграция и
  управление мастер-данными, Бюджетирование, Корпоративное казначейство, Процессы/задачи/оповещения,
  Управление договорами, Централизованное управление закупками и активами, **Инструменты сверки и
  элиминации ВГО**.
- Различающая сигнатура: **полное решение управления ГРУППОЙ поверх баз-источников (НЕ ведёт первичный
  оперативный учёт)**: консолидация + МСФО + казначейство + мастер-данные + ССП/инвестпроекты + элиминация ВГО.
- Негатив: vs ERP.УХ — отдельная база-сборщик без встроенного учётного ядра; vs Бухгалтерия КОРП МСФО — шире
  трансформации (управление всей группой); vs БИТ.Финанс — казначейство/бюджеты встроенные, не надстройка.
- Маршрут: `uh` → kb-uh-str/met. Уровни: 1,2,4,9,6. Источник: v8.1c.ru/cpm.

**1С:Бухгалтерия КОРП МСФО** · `key: msfo` · база (бух + параллельный МСФО)
- Что: бухгалтерский и налоговый учёт + **параллельная оценка по МСФО** для предприятий **БЕЗ дочерних
  обществ** (одна организация).
- Параллельный учёт по МСФО: ОС, НМА, активы для продажи, финансовые инструменты, лизинг, резервы
  (сомнительные долги, снижение стоимости запасов), отложенные налоги, себестоимость/финрезультат;
  **трансляция проводок** из подсистемы бух/налогового учёта в подсистему МСФО.
- Различающая сигнатура: **бухгалтерия одной организации + параллельный МСФО через трансляцию, БЕЗ
  консолидации группы** (для группы — УХ/ERP.УХ).
- Оговорка: отдельного продукта «1С:МСФО» в текущем каталоге нет. МСФО одной организации = «1С:Бухгалтерия
  КОРП МСФО»; МСФО группы = контур в УХ/ERP.УХ. Унаследованные «1С:МСФО»/«1С:Консолидация» — прежнего
  поколения (могут встречаться у заказчика).
- Маршрут: `msfo` → kb-msfo-str/met. Уровни: 2,3,1,9. Источник: v8.1c.ru/1c-buhgalteriya-msfo.

### D. Кадры и зарплата

**1С:Зарплата и управление персоналом (ЗУП, ПРОФ/КОРП)** · `key: zup` · база
- Подсистемы: Расчёт зарплаты, Кадровое делопроизводство, Учёт рабочего времени, Штатное расписание,
  НДФЛ/взносы/перс.учёт, Статотчётность.
- Объекты/сервисы: электронные трудовые книжки, заявление на отпуск, лицевой счёт, «Справочник кадровика»,
  1С:Кабинет сотрудника; ред. 3.1, платформа 8.3z (ФСТЭК).
- Различающая сигнатура: **ОТДЕЛЬНАЯ база ТОЛЬКО кадры+зарплата, БЕЗ бухучёта и производства, с обменом во
  внешний учётный контур**. Если зарплата встроена в одну базу с учётом → это КА/ERP, не ЗУП.
- КОРП-маркер: кадровый ЭДО, подбор, обучение, мотивация/грейды, охрана труда.
- Маршрут: `zup` → kb-zup-str/met; `zup-korp` → kb-zup-korp-str/met. Уровни: 2,3,5,9. Источник: v8.1c.ru/hrm.

### E. Логистика и склад

**1С:WMS Логистика. Управление складом** · `key: wms` · база (самостоятельная)
- Подсистемы/ядро: Приёмка→Размещение→Отбор→Отгрузка→Инвентаризация, Управление задачами ТСД.
- Термины-маркеры: **адресное хранение (ячейки/места), топология склада (зоны), задание на ТСД, штрихкодирование**.
- Различающая сигнатура: **адресный склад в реальном времени (ячейки+зоны+ТСД)**. Нет адресного хранения и
  ТСД → это складской блок типовой (УТ/ERP), не WMS. vs TMS — хранение, а не перевозки.
- Маршрут: `wms` → kb-wms-str/met. Уровни: 5,9,7,11. Источник: solutions.1c.ru/catalog/wms.

**1С:TMS Логистика. Управление перевозками** · `key: tms` · база (самостоятельная)
- Подсистемы: Управление транспортом, Планирование грузоперевозок (сборные/комплектные/мультимодальные),
  последняя миля.
- Объекты: «Грузоперевозка (рейс)», «Заявка на перевозку груза», «Привлечённый транспорт»; маршрутизация/тарификация.
- Маркеры: ред. 3.0 (релиз 3.0.9.x), 1С+ITOB, 1С-Совместно, Реестр росПО.
- Различающая сигнатура: **управление перевозками (рейсы/заявки/привлечённый транспорт/маршруты/тарифы/
  последняя миля), БЕЗ учётного ядра**. vs WMS — транспорт, а не адресное хранение.
- Маршрут: `tms` → kb-tms-str/met. Уровни: 5,9,4. Источник: solutions.1c.ru/catalog/tms.

### F. Мастер-данные, интеграция, аналитика (платформенный/внешний слой)

**1С:MDM Управление НСИ** · `key: mdm` · база (самостоятельная)
- Подсистемы: Управление НСИ, множественная классификация, группировки, база нормативно-технических
  документов, репликация в системы-потребители.
- Объекты/цепочка: **Заявка на изменение НСИ → эксперт → эталонный объект → репликация**; дедубликация,
  выравнивание атрибутов; ред. 2.5, совместно с 1С:ERP 2.
- Различающая сигнатура: **управление СОДЕРЖАНИЕМ справочников (дедуб/классификация/выравнивание + раздача
  эталонных мастер-данных)**. vs Шина — нормализует НСИ, а не перевозит сообщения; vs УХ — только справочные данные.
- Маршрут: `mdm` → kb-mdm-str/met. Уровни: 4,9,7,10. Источник: solutions.1c.ru/catalog/mdm.

**1С:Шина** · `key: shina` · внешний (ESB) · платформенный слой
- Класс: **Сервисная шина предприятия (ESB)** — обмен данными между ИС (1С и не-1С) через единую точку
  входа/выхода с единым мониторингом.
- Способы подключения: Веб-сервисы (SOAP/WSDL), JMS (ActiveMQ Artemis, SAP PI), AMQP (RabbitMQ), JDBC
  (MS SQL/PostgreSQL), Сервисы интеграции (механизм платформы 1С), обмен файлами (FTP), HTTP/REST, KAFKA.
- Функциональность/маркеры: Среда разработки в браузере (схемы интеграции, Gitlab), Панель управления
  (администратор), Мониторинг и контроль сообщений (доставленные/недоставленные, гарантированная доставка);
  на технологии «1С:Предприятие.Элемент»; поставки 100/500/без ограничений; реестр росПО.
- Различающая сигнатура: **единая точка обмена сообщениями (ESB) с мониторингом доставки**; перевозит
  сообщения, но НЕ нормализует НСИ (это MDM) и НЕ строит дашборды (это BI). Внешняя платформа над ландшафтом.
- Маршрут: `shina` → kb-shina-met. Уровни: 9,7,11. Источник: v8.1c.ru/static/1c-shina.

**1С:Аналитика** · `key: analytics` · внешний/платформенный (BI)
- Класс: **BI-система, ЧАСТЬ платформы «1С:Предприятие 8»** — общие учётки/права, переход из учётной
  системы «в пару кликов».
- Режимы: Рабочий стол (поиск/создание диаграмм и дашбордов), просмотр/редактирование диаграммы,
  просмотр/редактирование дашборда; детализация до документа/элемента, разрезы, регистры; мобильные (Android/iOS).
- Различающая сигнатура: **BI «вживую» поверх данных самой базы 1С, БЕЗ ETL/выгрузки и без отдельных
  учётных записей**. vs PIX BI — не консолидирует внешние источники и не выходит за пределы базы 1С.
- Маршрут: `analytics` → kb-analytics-met. Уровни: 1,7. Источник: v8.1c.ru/platforma/1s-analitika.

## §3. Слабые признаки — НЕ опознавать в одиночку

Эти признаки встречаются у многих продуктов; различают только в комбинации или через ОТСУТСТВИЕ:
- **Казначейство** — erp, ka, ut, bf, bitstr, erpuh, uh.
- **Бюджетирование / Управление финансами** — erp, ka, bf, bitstr, erpuh, uh.
- **Мониторинг/дашборды** — почти у всех крупных баз (+ pix, analytics как сам продукт).
- **CRM/Продажи/Закупки/Склад** — общий оперативный контур erp/ka/ut/unf (различает ОТСУТСТВИЕ).
- **Управление персоналом/зарплата** — erp, ka, upp, unf, bp (упрощённо), zup (отдельно).
- **Регламентированный учёт** — erp, ka, bp, upp, erpuh, uh (отделяет торговую ветку от учётной, не внутри).
- **Управление производством** — ключ в ГЛУБИНЕ (полное/облегчённое/учёт), не в факте.
- **МСФО / Консолидация** — erp, bf, erpuh, uh, msfo (не разделяет встроенное/надстройку/отдельное решение).
- **Платёжный календарь / Заявка на расход ДС** — bf, erpuh, uh, bitstr (различает «сверх типового + маркеры БИТ»).
- **ЦФО / Статьи оборотов / Сценарии** — bf, bitstr, erpuh, uh.
- **Платформа 1С:Предприятие 8** — почти у всех (различает конкретная версия 8.2 vs 8.3/8.3z).
- **Инвентаризация / Штрихкодирование** — типовые УТ/ERP/КА и WMS (для WMS нужны ячейки/ТСД).
- **Эталонная НСИ / мастер-данные** — mdm, erpuh, uh (различает механика: дедубликация/репликация vs консолидация).
- **Реестр российского ПО** — у многих, не различает вовсе.

## §4. Маршрутизация: product-key → доменные БЗ

Навигатор отдаёт **стабильный `product-key`**; пара доменных БЗ выводится по соглашению `kb-<key>-str`
(структура) и `kb-<key>-met` (методика). Надстройки — модификаторы, выбирают комбинированную БЗ:

| Опознано | product-key | Доменные БЗ |
|---|---|---|
| База без надстроек | `erp`/`ka`/`ut`/`bp`/`upp`/`unf`/`zup`/`zup-korp`/`uh`/`erpuh`/`wms`/`tms`/`mdm`/`msfo` | `kb-<key>-str` + `kb-<key>-met` |
| База + БИТ.Финанс | `<база>+bf` | `kb-<база>-bf-str` + `kb-<база>-bf-met` |
| БП + БИТ.Строительство + БИТ.Финанс | `bp+bs+bf` | `kb-bp-bs-bf-str` + `kb-bp-bs-bf-met` |
| Платформенный/внешний | `shina`/`analytics`/`pix-*` | `kb-shina-met` / `kb-analytics-met` / `kb-pix-*-met` |
