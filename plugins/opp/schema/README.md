# schema — описание устройства памяти

Здесь живёт **единое машинное описание** проектной памяти: какие есть таблицы фактов, какие у них
поля, какие связи между ними и какие инструкции заполнения. Формат — **YAML**.

Принцип «узкой талии»: это описание — единственный источник, из которого потом порождаются и
человеко-читаемая документация, и автоматические проверки (контролёр целостности). **Один список
префиксов** (кодов таблиц) — производный из этого описания, не ведётся отдельно.

## Состояние

Наполнено (компонент 04 — готово): `fact-tables.yaml` — машинный контракт **39 таблиц фактов**
(поля, рёбра, рецепты, дефолт-инструкции, целостность, стыки; паспорта — `spec/fact-tables/`);
`model.py` — парсер контракта в модель; `lint.py` — линтер контракта (шаг verify);
`extract.py` — задание на извлечение строк + upsert в память (cap-t-extract);
`instructions.py` — эффективная инструкция заполнения с per-project правкой и откатом (cap-instr).

> Способ хранения строк-данных (сами записи памяти) — отдельное решение Итерации 4; здесь описывается
> только *устройство*, не данные конкретного проекта.
